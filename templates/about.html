What is this website?

This website is a WEB UI for docker. It is mainly built with angularjs, jquery and Docker's API.

What is Docker?

Docker is an open platform for developers and sysadmins to build, ship, and run distributed applications. Consisting of Docker Engine, a portable, lightweight runtime and packaging tool, and Docker Hub, a cloud service for sharing applications and automating workflows, Docker enables apps to be quickly assembled from components and eliminates the friction between development, QA, and production environments. As a result, IT can ship faster and run the same app, unchanged, on laptops, data center VMs, and any cloud. ( Docker )

How can I test it?

In the active containers list, you will see running containers. There should be one running. If not, you can click to the "All containers" on the left , select one and click on "Manage". From there, you can click "Start" and go back to the "Active containers" page.
The interesting column is "Exposed Ports". It shows the container's local port and where it is redirected to, on the host. If you click the link, you will access the container's web Server.
From there, you can see a simple page. The content of the page has actually been pulled out from my own gitHub repository (click the link on the page to check it out) . Everytime the container starts, it pulls the repo out.

Can I try that myself?

Sure , but you will need a repo containing some files (ideally, html or php).
Click on "New" in the Containers section. Give your container a hostname, select gitWebServer as an Image and replace 'https://github.com/Your/repo.git' with your repo address. (leave '/run.sh ,')
Click on "Create", you will be redirected to the container's page.
In the Host IP field, type '0.0.0.0' , this means you want to listen on all the interfaces. In the Host Ports field, enter a port number (60090 for example).
Click on Start and go back to the Active Containers page.
Click on the link in the exposed ports columns corresponding to your container, if everything went well, you should see the files pulled from your repo !

Why this website?

The purpose of this website is to be able to create containers really quickly. You give it a name, select a pre-built image , give it a bunch of commands and you create it ! Possibilities with docker are endless and you can create your custom images (Apache, PHP, Python, node.js, ...) with the tools you need.
It can be useful for developers to test their app. No need for expensive VM's and hours of tools installation. Once your image is created with your tools and configured with your repo, you can test your work. If you update the repo, simply restart the container.
